## 🤖 Autopilot 执行场景（v0.2.24.0 新增）

| 场景 | 用户动作 | Agent 行为 |
|------|---------|-----------|
| **一键预订所有** | 点击"Book everything →" | 后台启动 3 个 headless 浏览器，分别操作 Kayak / Booking.com / OpenTable |
| **餐厅时段不可用** | Agent 自动处理 | 按 ±30/60/90 分钟扩散自动尝试备用时段，不打扰用户 |
| **首选酒店无房** | Agent 自动处理 | 自动切换 backup_plans 中的次优酒店，继续执行 |
| **网络错误** | Agent 自动处理 | 2s / 5s 间隔重试最多 3 次 |
| **所有手段都失败** | Agent 标记需人工介入 | 生成 Action Items + 手动预订链接，继续处理其他步骤 |
| **关闭页面继续执行** | 用户关闭浏览器 | keepalive fetch，后台继续，完成后推送通知 |
| **查看 Agent 决策过程** | 在 /trips 展开步骤 | 时间线格式展示每一个决策：尝试了什么、结果如何、为什么调整 |

## 📊 Learning Loop 场景

| 场景 | 系统行为 |
|------|---------|
| **方案采纳追踪** | 用户选了哪个方案？主方案还是备选？ |
| **满意度反馈** | 去了以后觉得怎么样？哪里不对？ |
| **偏好学习** | 用户总是选安静的 → 下次默认安静权重更高 |
| **异常预警** | 推荐的餐厅评价突然变差 → 提醒用户换方案 |
| **Agent 接受度** | 用户是否接受 agent 的时间/场馆调整 |
| **平台成功率** | OpenTable 比 Booking.com 成功率高 → 优先尝试 |
| **手动覆盖分析** | 哪些场馆 agent 选了用户最常换掉 → 未来降权 |

## 🗺️ 用户旅程示例

### 场景 A：一个人约会规划（传统模式）

```
用户: "下周五约会，曼哈顿，$80/人，安静"
                    ↓
Onegent: [解析意图] date_night + restaurant + bar
                    ↓
Onegent: [并行搜索] 餐厅 + 酒吧候选（Google Places + Tavily）
                    ↓
Onegent: [评分排序] 基于评论信号 + 用户偏好
                    ↓
Onegent: [组装方案] 主方案 + 2 备选 + 餐后去处 + 风险
                    ↓
用户: "换个便宜点的"
                    ↓
Onegent: [Refinement] 只更新价格约束，保留安静/Manhattan/日期
                    ↓
用户: [点击 Share]
                    ↓
Onegent: [保存方案] → 生成链接 → 复制到剪贴板
                    ↓
朋友: [打开链接] → 看到方案 → 点"This works for me"
                    ↓
Onegent: [记录反馈] partner_approved
                    ↓
用户: [点击 Book everything →]
                    ↓
Onegent: [Autopilot 启动] 后台 OpenTable 浏览器打开
                    ↓
OpenTable 7pm 无位 → Agent 自动尝试 7:30pm → 成功 ✓
                    ↓
推送通知: "🍽 Le Bernardin pre-filled at 7:30pm — open and pay"
                    ↓
用户: [点击通知] → /trips → 展开决策日志 → 点击 Open → 付款
```

### 场景 B：两人一起决策（Decision Room）

```
A: "安排一个周五约会，我不喜欢太吵的"
                    ↓
Onegent: [创建 Decision Room] → 生成分享链接
                    ↓
A: [发给 B] — "帮我选一下"
                    ↓
B: [打开链接] → 输入约束: "想吃意大利，不要等位"
                    ↓
Onegent: [合并约束] 安静 + 意大利 + 不等位
         [无冲突] → 生成 3 张候选卡
                    ↓
A 和 B 分别独立投票
                    ↓
双方都点了"Buccini" → "You both agreed on Buccini ✓"
                    ↓
[事后反馈] A: "Loved it" / B: "Fine"
```

### 场景 C：周末旅行完整执行

```
用户: "下个月去 Chicago 过周末，预算 $900"
                    ↓
Onegent: [并行] 航班 pipeline + 酒店 pipeline
                    ↓
Onegent: [配对组包] 3 套方案 + 时间衔接检查 + 总价
                    ↓
用户: "选套餐 A，帮我订"
                    ↓
Onegent: [Autopilot]
  ✈ Kayak → 找到 AA2341，返回 checkout 链接 ✓
  🏨 Booking.com → 导航到 Hotel X 预订页 ✓
  🍽 OpenTable → 7pm 无位 → 7:30pm 自动调整 ✓
                    ↓
推送: "✈ 🏨 🍽 Your Chicago trip is ready — 3/3 pre-filled"
                    ↓
用户: [/trips] → 看 Agent 决策日志 → 3 个 Open 按钮 → 逐一付款
```
